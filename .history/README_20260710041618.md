# 💊 Med Reminder Bot — напоминания о лекарствах и визитах к врачу

Полностью рабочий Telegram-микросервис на **Node.js + TypeScript** (grammY):

- Напоминания о приёме лекарств и визитах к врачу с кнопкой подтверждения
- Эскалация ("бот ругается"), если не подтвердить вовремя
- Уведомление ребёнка/наблюдателя о статусе родителя
- Поддержка пожилых людей **без Telegram** — напоминания по SMS (Twilio)
- Оплата **прямо внутри Telegram** (нативная кнопка "Оплатить"), провайдер — LiqPay (ПриватБанк):
  деньги приходят прямо на карту ПриватБанка, без Wise/Payoneer/Stripe
- Глобальный переключатель: сначала всё **бесплатно** (пока набираешь аудиторию), потом одной
  настройкой включаешь оплату 3€/мес

> React в этом проекте не нужен — это библиотека для интерфейсов сайтов, а Telegram-бот
> взаимодействует с пользователем через сообщения и кнопки самого Telegram. Правильный
> выбор для бота — именно то, что здесь и написано: Node.js + TypeScript.

## 1. Создание бота в Telegram

1. Открой Telegram, найди **@BotFather**
2. Отправь `/newbot`
3. Придумай имя (например: "Напоминалка о лекарствах") и username (должен заканчиваться на `bot`, например `pillcare_helper_bot`)
4. BotFather пришлёт токен вида `123456789:AAExample...` — сохрани его, он понадобится в `.env`
5. Дополнительно можно настроить командой `/setcommands` список команд для автодополнения:
   ```
   start - Начать работу с ботом
   help - Справка
   add_elder - Добавить подопечного (родителя, ребёнка или взрослого с инвалидностью)
   add_medication - Добавить напоминание о лекарстве
   add_appointment - Добавить напоминание о визите к врачу
   my_elders - Список подопечных
   status - Статус подписки
   subscribe - Оформить платную подписку
   ```

Нужно **два процесса одновременно** (в двух вкладках терминала):

# Терминал 1 — сам бот

показывать первым пользователям сколько угодно, без блокировок.

## 5. Подключение SMS для пожилых родителей без Telegram (Twilio) — необязательно

1. Зарегистрируйся на [twilio.com](https://www.twilio.com/) (есть бесплатный триал с тестовым балансом)
2. В консоли Twilio купи номер телефона (Phone Numbers → Buy a number) — для теста подойдёт бесплатный триальный номер
3. Скопируй `Account SID` и `Auth Token` с главной страницы консоли — впиши в `.env`:
   ```
   TWILIO_ACCOUNT_SID=...
   TWILIO_AUTH_TOKEN=...
   TWILIO_PHONE_NUMBER=+1xxxxxxxxxx
   ```
4. Чтобы Twilio мог присылать тебе ответы пользователей ("ПРИНЯЛ"), нужен публичный URL — см. шаг 9.
   Пропиши в настройках номера в Twilio:
   **Phone Numbers → твой номер → Messaging → "A message comes in" → Webhook:**
   `https://твой-домен/webhooks/twilio/sms`

> ⚠️ Пока не задан `TWILIO_ACCOUNT_SID`, бот просто не отправляет SMS и пишет предупреждение в консоль —
> остальной функционал не ломается. Функция нужна только для родителей вообще без Telegram.

## 6. Подключение оплаты (LiqPay внутри Telegram) — деньги приходят прямо на карту ПриватБанка

**Почему именно так:** Stripe официально не поддерживает создание аккаунтов для резидентов Украины
(проверено на актуальный момент). Зато у Telegram есть встроенная поддержка **LiqPay как официального
платёжного провайдера для Украины** в Bot Payments API — это значит, что человек платит прямо
в чате Telegram нажатием одной кнопки, а деньги идут через LiqPay (сервис самого ПриватБанка) прямо
на твой счёт/карту. Никаких отдельных сайтов, редиректов и веб-хуков для оплаты настраивать не нужно.

**Про Telegram Stars (встроенная валюта Telegram) — почему НЕ они:** это не то же самое, что "деньги
на карту". Вывод Stars идёт только через Fragment → криптовалюту TON → продажа на бирже → и только
потом на счёт, с заморозкой на 21 день и потерей ~15-30% на комиссиях App Store/Google Play при
оплате с телефона. Для цели "евро на карту ПриватБанка" LiqPay через обычные Telegram-инвойсы —
прямой и понятный вариант.

**Важно (это не юридическая или налоговая консультация, а только фактическая информация):**

- Для **регулярного** приёма платежей от клиентов LiqPay потребует регистрацию мерчант-аккаунта на
  **ФОП** (или юрлицо) — физлицу без предпринимательского статуса подключить приём повторяющихся
  платежей от клиентов, скорее всего, не дадут пройти модерацию.
- Для IT-услуг/подписочных сервисов обычно подходит **2 или 3 группа единого налога** — но точную
  группу, ставку и то, нужен ли ты плательщиком НДС при таких оборотах, лучше уточнить у бухгалтера
  или на consultant.tax.gov.ua — это зависит от годового оборота и вида деятельности (КВЕД).
- Деньги от клиентов в EUR при расчётах через LiqPay обычно поступают в **гривне** (конвертация по
  курсу на момент зачисления) — это нормальная практика для украинского эквайринга.

### Шаги регистрации и настройки:

1. Зайди на [liqpay.ua](https://www.liqpay.ua/) → **Регистрация** → тип "Юридична особа / ФОП"
   (нужен код ЄДРПОУ/РНОКПП ФОП и реквизиты счёта в ПриватБанке)
2. В личном кабинете: **Мої магазини → створити магазин** → укажи назначение (можно указать ссылку
   на Telegram-бота)
3. Дождись модерации (обычно 1-2 рабочих дня) — после неё в разделе **API** появятся `Публічний ключ`
   и `Приватний ключ`
4. В Telegram зайди в **@BotFather → /mybots → выбери своего бота → Bot Settings → Payments**
5. В списке провайдеров выбери **LiqPay**, следуй подсказкам BotFather — тебя попросят ввести
   публичный и приватный ключ LiqPay из шага 3
6. BotFather пришлёт **provider_token** (строка вида `12345:LIVE:AbCdEf...`) → впиши в `.env` как
   `TELEGRAM_LIQPAY_PROVIDER_TOKEN`

Готово — больше ничего настраивать не нужно, никаких публичных доменов или веб-хуков для оплаты.

### Как это работает технически (уже реализовано в коде)

- `src/services/paymentService.ts` — отправляет нативный Telegram-инвойс (`sendInvoice`) с суммой из `.env`
- `src/bot/callbacks/payments.ts` — обрабатывает подтверждение оплаты (`pre_checkout_query`) и
  успешную оплату (`successful_payment`), продлевает подписку на месяц
- Раз в месяц (за 1 день до окончания периода) воркер сам присылает пользователю новый инвойс на
  продление — это реализовано через очередь `billing-reminder` в `src/scheduler/queues.ts`

### Включение оплаты (когда наберётся аудитория)

В `.env` есть переключатель:

```
BILLING_ENABLED=false
```

Пока `false` — бот **бесплатен для всех**, без ограничений и без пробного периода. Когда решишь
начать зарабатывать — поставь `true` и перезапусти бота (`pm2 restart medbot-server medbot-worker`
на проде, либо просто перезапусти `npm run dev` локально). С этого момента новым пользователям даётся
3-дневный пробный период (`TRIAL_DAYS=3`), после которого бот попросит `/subscribe`.

Цена подписки задаётся в `.env`:

```
SUBSCRIPTION_PRICE_EUR=3
SUBSCRIPTION_CURRENCY=EUR
```

## 7. Как выглядит сценарий использования (проверь сам)

1. Ребёнок открывает бота → `/start`
2. `/add_elder` → вводит имя мамы → бот спрашивает "есть ли у неё Telegram"
   - Если да — просит связать по Telegram ID (мама тоже должна сделать `/start` у этого бота)
   - Если нет — просит номер телефона → напоминания пойдут по SMS
3. `/add_medication` → выбирает маму → название лекарства → дозировка → время (например 09:00, 21:00)
4. В указанное время маме приходит красивое анимированное сообщение с кнопкой "Я принял(а) лекарство"
5. Если мама не нажимает кнопку 15 минут — бот присылает повторное, более настойчивое сообщение, и
   одновременно пишет ребёнку "мама ещё не подтвердила"
6. Если мама нажимает кнопку — ребёнку приходит "мама подтвердила вовремя"
7. Аналогично работает `/add_appointment` для визитов к врачу (напоминание за 2 часа по умолчанию)
8. Когда включишь `BILLING_ENABLED=true` — по истечении пробного периода `/subscribe` пришлёт
   нативную кнопку "Оплатить" прямо в чате

## 8. Связка "опекун — подопечный" и доступность для людей с инвалидностью

В коде используется термин "elder" (и в текстах бота — "подопечный"), но это не привязано к возрасту:
подопечным может быть пожилой родитель, ребёнок, или взрослый человек с инвалидностью любого возраста —
логика для всех одна и та же:

- **Опекун (ребёнок/родственник)** пользуется своим Telegram, добавляет подопечного через `/add_elder`,
  настраивает лекарства и термины
- **Подопечный** не обязан ничего устанавливать или регистрироваться:
  - Если он умеет пользоваться Telegram — напоминания приходят ему туда с кнопкой подтверждения
  - Если не умеет (или физически не может — например, из-за нарушений зрения/моторики) — напоминания
    идут простым **SMS**, а подтвердить можно **любым коротким ответом**: словом "ПРИНЯЛ"/"ИДУ" или
    просто цифрой **1** — это сделано специально, чтобы не требовать точной формулировки
- Один подопечный может иметь **нескольких опекунов** (например, несколько детей одновременно) —
  каждый добавивший его через `/add_elder` будет получать уведомления о статусе
- Один опекун может добавить **сколько угодно подопечных** — родителей, детей, других родственников —
  и переключаться между ними при добавлении лекарств/термин через `/add_medication` и `/add_appointment`

### Что уже учтено для доступности

- SMS-подтверждение принимает много вариантов ответа (см. `src/services/smsService.ts`,
  функция `isConfirmationText`) — не нужно помнить точное слово
- Сообщения в Telegram всегда содержат текстовое описание (caption), даже если есть анимация —
  это работает с экранными считывающими программами (screen readers) для незрячих пользователей
- Кнопка подтверждения — это всегда один явный текстовый вариант ("✅ Я принял(а) лекарство"), без
  скрытых жестов или сложных взаимодействий

### Что можно добавить дальше, если понадобится (пока не реализовано)

- Голосовое SMS/звонок вместо текстового SMS (актуально при нарушениях зрения совместно с неумением
  читать SMS) — потребует IVR-интеграцию через Twilio Voice
- Увеличенный шрифт/упрощённый текст для людей с когнитивными особенностями — сейчас текст уже
  короткий и простой, но можно сделать отдельный "упрощённый" шаблон сообщений

## 9. Публичный URL (нужен только для Twilio SMS, необязательно для оплаты)

Для **самого Telegram-бота** и для **оплаты** публичный домен не нужен вообще (бот работает через
long polling, а оплата — через нативные Telegram-инвойсы). Публичный URL нужен только если ты
подключаешь SMS для родителей без Telegram (шаг 5).

**Для теста — быстрый вариант через ngrok:**

```bash
npm install -g ngrok
ngrok http 3000
```

Скопируй выданный `https://xxxx.ngrok-free.app` и пропиши его в `.env` как `PUBLIC_URL`, а также
используй в настройках Twilio webhook URL (шаг 5).

Рекомендуемый недорогой вариант — VPS (например, Hetzner CX22, ~4-5€/мес):

# на сервере

## 11.5 Курсы приёма, план и доступность (новое)

### Курс приёма лекарства

При добавлении лекарства (`/add_medication`) теперь спрашивается **длительность курса в днях**:

- введите число дней (например `14`) — бот вычислит дату окончания `endDate` и автоматически
  **деактивирует лекарство** по завершении курса (повторяющаяся задача BullMQ снимается);
- введите `0` — приём **постоянный** (без срока).

### Счётчик дней

В каждом напоминании и в плане виден прогресс курса:

- **День X из Y** — сколько дней прошло с начала курса;
- **подтверждено N дн.** — сколько уникальных дней фактически подтверждено (по нажатию кнопки / голосу / SMS).

### Полный план приёма (`/my_plan`)

Команда показывает на сегодня сгруппированный по времени список всех активных лекарств
подопечного с прогрессом по каждому. Если лекарств несколько в один день — они собираются
в один план по времени приёма.

### Доступность для незрячих и глухих (`/accessibility`)

У пользователя есть режим доступности (`AccessibilityMode` в БД):

- **Обычный** — текст + кнопки (как раньше).
- **Незрячий (BLIND)** — напоминания дублируются **голосовым сообщением (TTS)**, а подтвердить
  приём можно **голосовым сообщением** (любое входящее voice-сообщение подтверждает последнее
  неподтверждённое напоминание). Тексты строятся так, чтобы их корректно читал скринридер.
- **Глухой/слабослышащий (DEAF)** — акцент на **визуальное** (текст + анимация), без звука.

Голос генерируется провайдером TTS, настраиваемым через env (см. `.env.example`):

```
TTS_PROVIDER=openai
OPENAI_API_KEY=sk-...
TTS_VOICE=alloy   # опционально
```

Если провайдер не задан — голос не генерируется, уведомления отправляются обычным текстом (фолбэк).
